# Spec — Authentication

> Feature: Register / Login / Logout
> User Story covered: **US1**
> Branch: `main` (base feature, included from initial setup)
> Agent used: OpenCode

---

## Context

The application is **strictly private**: each user can only see their own domains and concepts. Authentication is the absolute prerequisite for all other features. Without a logged-in user, no data should be accessible.

---

## Goal

Set up a complete authentication system with:
- Registration (name, email, password)
- Login (email + password)
- Logout
- Route protection for all private pages

---

## What I WANT

- Use **Laravel Breeze** (Blade + TailwindCSS) — not Jetstream, not Livewire
- All routes protected by the `auth` middleware **except** login and register
- Redirect after login to `/dashboard`
- Redirect after logout to `/login`
- The registration form must ask for: `name`, `email`, `password`, `password_confirmation`
- Server-side validation with clear per-field error messages

---

## What I DO NOT WANT

- No email verification (disable email verification)
- No "remember me" checkbox
- No password reset for v1 (out of scope)
- No third-party auth packages (Socialite, Passport, Sanctum)
- No generic error messages — each field must have its own specific message

---

## Expected File Structure

```
app/Http/Controllers/Auth/
├── RegisteredUserController.php   (generated by Breeze)
├── AuthenticatedSessionController.php
└── ...

resources/views/auth/
├── login.blade.php
├── register.blade.php
└── ...

routes/
├── auth.php     (Breeze routes)
└── web.php      (routes protected by auth middleware)
```

---

## Business Rules

1. A user can only see domains they created themselves
2. All routes under `/domains`, `/concepts`, `/dashboard` must use `->middleware('auth')`
3. The `User → Domain` relationship is `hasMany` — `user_id` is required on every domain
4. When creating a domain or concept, always save `auth()->id()` or use `auth()->user()`


- [ ] Register with an already taken email → error message on the email field
- [ ] Register with non-matching passwords → error message
- [ ] Login with correct credentials → redirected to `/dashboard`
